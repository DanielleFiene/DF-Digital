DokData - Geospatial Search & Analysis Features

Frontend development for traffic analytics platform

Built advanced search, filtering, and data visualization components for a geospatial web application used for road safety analysis across Europe.

Role: Frontend Developer (freelance)

Timeline: May 2024 - February 2025

Tech: React, TypeScript, Mapbox GL JS, Leaflet, Turf.js

What I built:

  • Smart geocoding search: Address/street/highway search across Europe with intelligent API routing (Photon for addresses, Nominatim for highways). Handles 18+ European languages with street name normalization, diacritic folding, and multiple postcode formats. Multi-tier caching (localStorage + in-memory with TTL) for improved performance.
  • Street View optimization: Solved a performance issue where the previous implementation loaded all 100 Street View images immediately when the top 100 table was displayed. Implemented lazy-loading pattern where images only load when the user clicks the dropdown, reducing initial API calls from 100+ to zero. Each image loads on-demand with a single API call per request, significantly improving page load performance and reducing API costs.
  • Advanced Excel export: Built export system that generates spreadsheets with embedded Street View images (parallel loading with 4 concurrent requests, base64 conversion), map screenshots, calculated risk scores, and dynamic formulas using ExcelJS.
  • Map style switching: Implemented basemap selector that preserves custom data layers when switching between street/satellite/dark modes, with dynamic color adaptation for optimal visibility.

Code is proprietary and under NDA. Technical details available on request.
